# Session 7 — Git Branches and Documentation

Today I practiced a more realistic Git workflow.

## What I practiced

- Cloning a GitHub repository
- Checking the remote URL
- Creating a .gitignore file
- Creating folders for better repo organization
- Creating and switching branches
- Making documentation changes on a branch
- Committing changes
- Merging a branch back into main
- Pushing changes to GitHub

## What I learned

Branches allow engineers to work on changes safely before merging them into the main codebase.

.gitignore helps prevent temporary files, logs, secrets, and environment files from being committed.

Good documentation makes a GitHub repo easier to understand and shows proof of learning.

## Commands I used

- git clone
- git status
- git remote -v
- git branch
- git switch -c
- git add
- git commit
- git merge
- git push
